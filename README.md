# 旅行行程单页模板 · Trip Itinerary One-Pager

一个**单文件、零构建、零后端**的行程网站模板:所有内容都在一个 `index.html` 里,改几个数据数组就能生成你自己的行程站。手机优先、中文界面,可「添加到主屏幕」当 App 用。

本仓库自带一份 4 城 9 天的西班牙 + 葡萄牙行程作为完整示例(巴塞罗那 → 塞维利亚 → 波尔图 → 里斯本;**行程为虚构演示数据**,住宿名称、地址与日期均为示例占位,非任何真实出行安排)。

## 快速开始

- **本地预览**:直接双击用浏览器打开 `index.html`(地图与图片需联网)。
- **在线部署**(任选其一):
  - GitHub Pages:仓库 Settings → Pages → Branch 选 `main` / root,稍等片刻访问 `https://<用户名>.github.io/<仓库名>/`
  - Netlify Drop:把 `index.html` 拖进 [app.netlify.com/drop](https://app.netlify.com/drop),立刻得到链接
- 无需任何 API key:地图用 Leaflet + OpenStreetMap,景点图片与简介来自维基百科公开接口,自动加载。

## 功能

- 📅 按天行程卡片:景点配图(点开看大图 + 维基百科中文简介)、门票 / 开放时间 / 购票方式、距离提示
- 🗺️ 每城总览地图,**不同颜色 = 不同天**,点标记看景点,一键生成当天 Google Maps 路线
- 🏠 住宿汇总、🚄 城际交通票务状态(已买 / 待定)、📝 按优先级排序的待办清单
- 🚉 「怎么到酒店」:步行 / 打车 / 公共交通三种方式对比
- 🌤️ 各城天气 & 穿衣建议、🍽️ 当地特色美食卡、🗣️ 当地常用语(带中文谐音,把屏幕给店员看也行)
- 顶部城市条吸顶、点击直达对应城市

## ✨ AI 穿搭插画 & 实时天气

- **「👗 穿搭」标签**:给行程的每一天配一套穿搭,支持自评与导出穿搭卡。上传的照片只保存在你自己的浏览器本地(IndexedDB),不经过任何服务器;Day 1 预置了一张 AI 生成示例(通用卡通形象)。
- **「🌤️ 天气」标签**:自动拉取各城市近 3 日实时预报(Open-Meteo,免 API key、纯前端调用)。
- 示例玩法:把衣服平铺照片 + 目的地城市喂给图像模型(如 Gemini 2.5 Flash Image),即可批量生成下面这种「穿搭 × 城市手账」插画(图中为 AI 生成的通用卡通形象):

## 📍 足迹打卡 & 行中勾选(旅行中使用)

- **「📍 足迹」标签**:旅行中一键打卡——自动记录时间与 GPS 位置(浏览器定位,需 https 环境),每条可补 3 张照片、一句话心得和心情;顶部城市连线图随打卡自动「点亮」,打开页面时还会静默补记位置点,行程结束导出 JSON 即可画出旅行轨迹。演示版预置了几条示例打卡,可直接删改体验。
- **行程页每个景点下有 `○ 完成 / ⏭ 跳过 / 📝 备注` 按钮**,每天底部可「➕ 临时加点」:旅行中随手勾选、和计划对照,勾「完成」会自动在足迹里生成一条记录(两个标签联动)。
- 所有打卡 / 勾选数据只存你自己的浏览器(IndexedDB / localStorage),**不经过任何服务器**;「导出」生成 JSON 便于行后汇总,「完整备份 / 恢复」用于换设备。

## 改成你自己的行程

数据和页面在同一个文件里:打开 `index.html`,找到 `<script>` 开头的几个数组,照着现有格式填即可,不需要懂前端。

| 数组 | 是什么 | 主要字段 |
|---|---|---|
| `CITIES` | 顶部城市条 | `nm` 城市名 · `mt` 显示的日期段 · `c` 城市主题色 |
| `FOOD` | 每城美食卡 | `n` 名称 · `wiki` 中文维基词条 · `en` 英文词条(用于自动配图)· `d` 一句话介绍 |
| `WEATHER` | 天气 & 穿衣 | `city` · `date` · `hi`/`lo` 温度 · `sum` 天气概况 · `wear` 穿衣建议 |
| `PHRASES` | 当地用语 | `zh` 中文 · `loc` 当地语 · `say` 中文谐音 |
| `DAYS` | **核心:每天的行程** | `d` 日期 · `city` · `title`/`tag` 标题 · `transport` 当天交通 · `stay` 住宿 · `spots` 景点列表 · `opt` 备选景点 |
| `STAY` / `MOVE` / `TODO` | 住宿汇总 / 城际交通 / 待办 | 见文件内示例 |

景点(`spots` 里每一项)的字段:`n` 名称 · `q` Google 地图搜索词 · `wiki` 中文维基条目 · `en` 英文维基条目(取图用)· `d` 介绍 · `a` 地址 · `dist` 距离提示 · `lat`/`lng` 地图坐标 · `ticket` 门票 · `hours` 开放时间 · `book` 购票方式。

### 偷懒做法:让 AI 帮你填 🤖

把整个 `index.html` 发给 Claude / ChatGPT 等 AI 助手,再附一段话:

> 以这个 HTML 为模板,保留全部功能和样式,帮我做一份新行程:
> - 日期:X月X日–X月X日,共 N 晚
> - 城市与住宿:(城市 / 住几晚 / 住宿名或区域)
> - 城际交通:(日期 / 路线 / 方式 / 是否已购)
> 景点、地图坐标、门票开放时间、当地美食、天气穿衣、常用语请帮我补全。

几分钟就能得到一份可直接部署的新行程页。
